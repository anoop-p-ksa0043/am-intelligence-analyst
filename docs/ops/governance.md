# Governance

## Ownership
- Product owner: roadmap and release sequencing
- Strategy owner: account model, normalization, fit logic
- Data steward: duplicate review and alias management
- Product operations / PMM: taxonomy and rule ownership
- Security / privacy owner: access, retention, and sensitive-data policy

## Cadence
- Product and rule review: bi-weekly or monthly
- Duplicate review queue: continuous with weekly audit
- Security and privacy review: quarterly or before sensitive module activation

## Release Gates
- Canonical identity rules approved
- Normalization schema approved
- Product taxonomy versioned and owned
- Validation workflow agreed
- Access model defined
- Refresh and staleness rules documented

## Escalation
- Conflicting installed-base logic
- Sensitive-data boundary questions
- Low-confidence recommendations proposed for broad rollout
- Source ingestion that violates approved-public-source posture
